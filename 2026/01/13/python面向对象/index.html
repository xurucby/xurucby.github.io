1. 概述

1.1 面向过程

  1. 定义：分析出解决问题的步骤，然后逐步实现。
    • 示例：婚礼筹办需依次完成请柬准备、宴席安排、仪式策划等步骤。
  2. 公式：程序 = 算法 + 数据结构
  3. 优点：所有环节、细节可自主掌控。
  4. 缺点：需考虑全部细节，工作量大。

1.2 面向对象

  1. 定义：找出解决问题的相关对象，然后为其分配职责。
    • 示例：婚礼筹办可委托摄影公司负责请柬、酒店负责宴席、婚庆公司负责仪式。
  2. 公式：程序 = 对象 + 交互
  3. 优点：
    • 思想层面：可模拟现实情景，更贴近人类思维，便于梳理和分析问题。
    • 技术层面：高复用（封装重复代码）、高扩展（新增功能不修改旧代码）、高维护（代码逻辑清晰）。
  4. 缺点：学习曲线较陡峭。

2. 类和对象

  1. 抽象：从具体事物中抽离共性、本质，舍弃个别、非本质特征的过程。
    • 示例：手机的共性数据（品牌、价格、颜色）和行为（通话）。
  2. 类：一个抽象的概念，对应生活中的“类别”。
  3. 对象：类的具体实例，即归属于某个类别的“个体”。
  4. 类是创建对象的“模板”。

3. 三大特征

3.1 封装

3.1.1 数据角度

  1. 定义：将一些基本数据类型复合成一个自定义类型。
  2. 优势：
    • 将数据与对数据的操作相关联。
    • 代码可读性更高（类是对象的模板）。

3.1.2 行为角度

  1. 定义：向类外提供必要的功能，隐藏实现细节。
  2. 优势：简化编程，使用者无需关注实现细节，仅需调用对外功能。
  3. 私有成员：
    • 作用：屏蔽无需向类外提供的成员。
    • 做法：命名使用双下划线开头。
    • 本质：名称被修改为_类名__成员名，可通过__dict__查看。

3.2.4 继承定义

  1. 概念：重用现有类的功能，并在此基础上扩展新功能。
  2. 相关知识：
    • 父类（基类、超类）、子类（派生类）。
    • 父类更抽象、范围更广；子类更具体、范围更窄。
    • 单继承：父类只有一个（如Java、C#）。
    • 多继承：父类有多个（如C++、Python）。
    • Object类：Python中所有类直接或间接继承自object类。

3.3 多态

3.3.1 定义

  1. 字面意思：同一种行为有不同表现形态。
  2. 概念：父类的一个方法，在不同子类中有不同实现。
  3. 说明：编码时调用父类方法，运行时传递子类对象，执行子类的重写方法。

3.3.2 重写

  1. 定义：在子类中定义与父类同名的方法。
  2. 作用：改变父类行为，体现子类个性。

3.4 设计思想

3.4.1 封装

  1. 思想：分而治之，变则疏之。
  2. 解释：将需求拆分到多个类中实现，变化点单独封装。
  3. 示例：创建人类、手机类、座机类，各自负责自身功能。

3.4.2 继承

  1. 缺点：通过继承复用代码时，新增功能需修改客户端代码。
  2. 思想：统一行为，隔离变化。
  3. 解释：父类统一方法接口，子类实现具体逻辑，客户端调用不受变化影响。

3.4.3 多态

  1. 思想：编码时调用父类方法，运行时执行子类方法。
  2. 解释：客户端按父类接口编程，子类重写方法，新增子类不影响客户端。
  3. 案例：老张出行（支持开车、坐飞机等多种方式）。
